Lifelines

Three lifelines — 50/50, Ask IT, and Skip — can each be used once before they refill. Multiple lifelines can be used on the same question (e.g. 50/50 then Ask IT) but each type is one-shot until refill.

Theme — colours

Changes apply live to this admin page (preview). Click Save at the top of this tab to push to all screens. Background, accent, text, success and danger are the most visible. The variants (mid / light / bright / dim) drive gradients and shadows.

Settings

Operational tools for managing the question bank, integrations, and developer-facing import/export options. Most operators never need to touch these — Browse sample packs (in the Questions tab) and CSV import/export should cover normal use.

Session control

Close session hides the join page from new visitors and rejects new player joins. Existing players keep their results view. Admin sign-in still works so you can reopen later. This is reversible and recommended over Shutdown for end-of-event hygiene.

Status: Open

▸ Developer options advanced

Raw JSON access to the question bank and a reset-to-defaults shortcut. JSON exports preserve the exact on-disk schema including bonus questions and per-question images; CSV is friendlier for spreadsheet editing.

Import JSON replaces the entire bank with the uploaded file's { main, bonus } arrays. Export JSON downloads the current bank in the same shape. Load bundled samples overwrites with the 10 main + 5 bonus phishing questions that ship with the app.

Process control

Trigger an in-process exit. Most hosting platforms (Railway, Fly, Render, Docker with --restart) auto-respawn the server within seconds, so this is closer to a rolling restart than a true off-switch. For a real offline state, scale your service to zero in the hosting dashboard. Use Close session above for the everyday "wrap up an event" flow.

Both rate-limited to one per 30 seconds per admin session. Restart tells clients to reconnect; Shutdown tells them the session has ended and stops their reconnect loop.

Maintenance — Export & Wipe Data

End-of-event reset. Downloads a final XLSX of the current results, then deletes data/config.json + data/questions.json and resets the in-memory game state. After this, the next operator joining will see a fresh setup with the deployment's default branding and an empty question bank.

Refused mid-game. Only available in the lobby or after the game has ended. Player connections are closed as part of the wipe; warn participants first.
